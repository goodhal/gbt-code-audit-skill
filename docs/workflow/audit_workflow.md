# 详细审计流程

本文档详细描述 LLM 审计的执行流程和验证机制。

## LLM 审计核心原则

### 独立性（最高优先级）

**❌ 绝对禁止**：
1. 查看快速扫描结果（findings/quick_scan/ 或 findings/baseline/）
2. 参考基线发现（baseline/目录内容）
3. 知道每个文件已有多少个发现
4. 使用"补充""另外""还有""共 X 个""快速扫描已发现"等暗示参考的词汇
5. 在 LLM 审计文件中提及快速扫描
6. 说"重点关注快速扫描可能遗漏的问题"（这暗示了你知道快速扫描的结果）
7. 自我设限审计范围（如"只关注业务逻辑漏洞"）

**✅ 必须执行**：
1. **完全忽略快速扫描的存在**，像"第一次看到这份代码"一样进行审计
2. 从零开始逐行阅读源代码
3. 独立判断每个安全问题
4. **全面覆盖所有类型的安全问题**，包括但不限于：业务逻辑漏洞、权限绕过、竞争条件、注入漏洞、认证授权问题等
5. 接受结果可能与快速扫描重复（会被 finalize_report 去重）
6. 结果数量完全不确定（可能 0 个，可能 20 个）

**💡 正确表述示例**：
- ✅ "独立发现的安全问题"
- ✅ "逐行分析源代码后发现"
- ✅ "代码中存在以下安全问题"
- ❌ "补充快速扫描未发现的漏洞"
- ❌ "除了快速扫描发现的问题外"
- ❌ "重点关注业务逻辑漏洞"（自我设限）

## 行号验证流程

```
┌─────────────────────────────────────────────────────────────────────┐
│                    LLM 审计行号验证流程                              │
├─────────────────────────────────────────────────────────────────────┤
│  1️⃣ 读取源文件  ◀ 【必须】使用 Read 工具读取完整源文件              │
│  2️⃣ 定位问题代码  ◀ 【必须】在源文件中精确搜索问题代码片段          │
│  3️⃣ 确认行号  ◀ 【必须】验证行号对应的是代码行而非注释行            │
│  4️⃣ 记录发现  ◀ 【必须】行号验证通过后才创建 md 文件               │
└─────────────────────────────────────────────────────────────────────┘
```

**⚠️ 强制要求**：
- **必须逐行核对源文件确认漏洞位置**
- **行号必须精确对应问题代码所在行**
- **禁止根据注释推断行号，必须定位实际代码**
- **每个漏洞的行号必须唯一且准确**
- **必须使用 Grep 工具精确查找行号，禁止凭记忆填写**

## 防幻觉工作流程

```
┌─────────────────────────────────────────────────────────────────────┐
│                    LLM 审计防幻觉流程                               │
├─────────────────────────────────────────────────────────────────────┤
│  1️⃣ 创建md文件  ◀ LLM写入审计发现到 findings/llm_audit/XXX.md     │
│  2️⃣ 调用验证    ◀ python skill.py validate_finding XXX.md         │
│  3️⃣ 验证结果    ◀ 成功 → 编号+1，开始下一个发现                    │
│               ◀ 失败 → 尝试修正行号 → 成功则通过，失败则用下一个问题覆盖 → 再次验证 │
└─────────────────────────────────────────────────────────────────────┘
```

**⚠️ 必须逐个验证**：
- **❌ 禁止批量验证**：创建多个md文件后一次性验证是错误的
- **✅ 必须一个一个验证**：创建一个md → 验证 → 通过后再创建下一个
- **✅ 验证通过才能继续**：只有验证成功后才能开始下一个发现

**⚠️ 验证失败处理**：
- **第一步**：尝试修正行号，使用Grep重新搜索正确的代码位置
- **第二步**：如果找到正确行号，更新md文件后再次验证
- **第三步**：如果无法找到正确行号，用下一个问题覆盖当前md文件内容后再次验证
- **禁止**：反复修改同一个问题超过2次，超过则跳过该问题

## 常见幻觉问题及预防

| 幻觉类型 | 错误示例 | 正确做法 |
|---------|---------|---------|
| 注释行号 | 报告行号27，实际是注释行 | 必须定位到实际代码行 |
| 行号偏移 | 报告行号150，实际代码在133 | 使用Grep精确搜索 |
| 代码不存在 | 报告的代码片段在源文件中不存在 | 必须在源文件中验证存在 |
| 重复行号 | 多个漏洞使用相同行号 | 每个漏洞必须有唯一行号 |
